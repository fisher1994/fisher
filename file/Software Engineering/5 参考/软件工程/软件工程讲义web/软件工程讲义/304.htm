第三章 需求分析和规格说明

3.3 数据流图（DFD）（续）

数据流图的两种画法：

1．由外向里画

    • 对于比较简单的系统，可以一气呵成画在一张纸上
    • 先画出系统的边界，即系统输入和输出
    • 然后画系统内部，即用一连串加工把系统的输入与输出联结起来，在数据流每次发生改变的地方画上一个加工，在数据需要存储的地方画上一个文件
    • 如果某个加工内部仍有数据流产生，应该将此加工用几个子加工代替
    • 注意：数据流是相互关联的、可以作为整体对待的一组数据

2．由顶往下画（分层数据流图）：

    • 比较复杂的系统不能画在一张纸上
    • 逐层分解的画法可以控制每一层的复杂度
    • 顶层：将整个系统作为一个加工，描述系统边界（输入与输出）
    • 中间层：表示某个加工分解为一组子加工，其中的子加工还需进一步分解
    • 底层：由不再进行分解的基本加工组成

3．注意事项：

    • 数据流和加工要适当命名
    • 父图与子图的平衡：子图的输入输出数据流应该与父图中相应加工的输入输出数据流一致
    • 局部文件：隐含在加工内部的文件，直到成为子加工之间的界面时才画出来
    • 分解的程度：不要一次分得太细，层数也不应太多
