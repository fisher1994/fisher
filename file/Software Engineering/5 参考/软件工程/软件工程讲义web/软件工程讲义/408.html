第四章 设计方法
4.10 结构化程序设计(SP)方法

1. 程序结构定理：任何程序逻辑都可以用顺序、选择、循环三种基本结构表示出来。

2. SP方法的特点：
    • 根据结构定理由顶往下逐步分解；
    • 用三种基本结构的组合表示模块的内部执行过程；

4.11 详细设计的描述方式

1. 程序流程图（FC）：
    • 方框表示加工；
    • 菱形框表示逻辑条件；
    • 箭头表示控制流（执行顺序）；
    • 结构化流程图：只用三种基本结构组合。

    2. 盒图（NS图）：

    • 五种基本成分（图4.61）；
    • 上入下出，限制随意跳转；
    • 非标准结构（如 FOR 循环语句）必须通过转换。

         3. 问题分析图（PAD）：

    • 三种基本成分（图4.64）；；
    • 可以用机械的规则进行编程（走树）（图4.65, 4.68）
