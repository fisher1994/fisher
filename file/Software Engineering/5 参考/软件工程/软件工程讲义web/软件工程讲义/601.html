第六章 检验与测试
6.1 基本概念

问题：
    • 软件中必然存在错误
    • 软件中的错误可能产生灾难性后果

解决：
    • 提高软件可靠性并且对软件进行严格的检验

检验手段：
    • 动态检查（白盒法、\盒法）
    • 静态检查（人工评审、读程序）
    • 程序证明（归纳法）

结论：
    • 检验的目的是证明错误的存在
    • 动态检查和静态检查都不可能发现所有的错误
    • 程序证明方法不适用于大型软件系统
    • 软件质量主要是在开发阶段决定的，不能靠检验手段去保证

6.2 软件评审（静态检查）

由人工审阅各种开发文档和源程序。
    • 软件开发的每一个阶段结束前都要进行评审
      •
    • 参加评审的人员与开发人员互相独立
    • 在评审条款中列出检查要点
    • 程序走查：模拟运行检查程序逻辑
    • 效果显著：30％－70％

6.3 测试（动态检查）

测试是为了发现错误而针对某些选定的输入（测试用例）运行程序，测试的关键在于设计一组“高产”的测试用例。

测试工作的难度之大出乎意料：例如下面的一段程序：

function Triangle(int a,int b, int c)
      If (a+b>c & b+c>a & c+a>b) Then
            If (a=b | b=c | c=a) Then
                  if a=b=c Then printf( '等边三角形' );
                  else   printf( '等腰三角形' );
           else  printf( '任意三角形' );
      else printf( '非三角形' );
Return

在选择测试用例时至少应考虑13种情况（pp.145-146）。

测试用例的基本特征：
    • 测试用例包括输入数据和预期输出结果两部分
    • 输入数据包括合理输入和不合理输入两种情况
    • 输出结果应检查除了应做的工作外，是否还做了不该做的事
    • 测试用例应该保留到系统生命期终止
