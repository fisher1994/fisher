第三章 需求分析和规格说明

3.3 数据流图（DFD）

数据流图的作用：

    • 描述系统的分解

数据流图的基本成份：

1．数据流

    • 符号：带方向（箭头）的线段或弧
    • 组成：一组相关联的数据
    • 命名：表达数据流的含义
    • 注意：区别控制条件、激发条件与数据流

2．加工：

    • 符号：圆
    • 作用：表示对数据所进行的操作
    • 命名：表示加工的作用
    • 编号：表示加工在分层数据流图（即对应子图）中的位置
    • 编号规则：（子图号）．（子图内部编号）

如：1，2，3，…

1．1， 1．2， 2．3， 3．3，…

1．1．2， 1．2．2， 2．3．1， 3．3．3，…

3．文件：

    • 符号：线段
    • 意义：存储的数据；读文件数据流朝外，写文件数据流向里
    • 命名：文件名与数据流名相同
    • 流入或流出文件的数据流名字可以省略
