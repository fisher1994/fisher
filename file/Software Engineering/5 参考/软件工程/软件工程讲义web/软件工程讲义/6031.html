6.5 黑盒法

不管程序内部逻辑，仅根据程序的功能说明来设计测试用例。

黑盒法有以下几种：
    • 等价分类法。
    • 边缘值分析法。
      •
    • 因果图法。
    • 错误推测法。

1. 等价分类法：将输入数据的可能值分成若干“等价类”，每类中取一个代表性的值。步骤如下：
    • 划分等价类：找出程序的输入条件，为每个输入条件划分出两个或以上等价类：包括合理与不合理等价类。（P.157）
      •
    • 选择测试用例，可分三步进行：

1）为每个等价类编号；
2）设计测试用例使之尽可能包括最多的尚未被其它测试用例包括的合理等价类，重复下去直到所有合理等价类全部被包括；
3）设计测试用例使之包括恰好一个尚未被包括的不合理等价类，重复下去直到所有的不合理等价类已被包括。

    2. 边缘值分析法：检查系统在输入或输出等价类的边界上的情况。

    与等价分类法比较：

    等价分类法只考虑输入条件，边缘值分析法同时考虑输入和输出条件；等价分类法从等价类中任选一个测试用例，边缘值分析法从等价类的边界周围选择测试用例。例如：

    • 考虑值的范围：检查恰好取到边界值的例子（合理输入）和刚刚超过边界值的例子（不合理输入）；
    • 考虑输入及输出数据个数：检查最小及最大个数（合理输入）和最小个数－1及最大个数＋1的情况（不合理输入）；
    • 对于有序集合：重点检查集合的头和尾的情况。
