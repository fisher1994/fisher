1．3 软件质量评价

软件质量的基本标准：

    • 可维护性（可读性、可修改性、可测试性）
    • 可靠性（正确性和健壮性－容错性）
    • 可理解性（逻辑简单、思路清晰）
    • 效率（执行速度快、占用资源少）

1．4 软件开发方法和自动工具

软件开发方法是指导软件研制的规范化过程：

    • 明确的开发步骤（工序化）
    • 具体的描述方式（文档格式、图形符号）
    • 确定的评价标准（对不同的设计方案作比较）

软件开发方法的例子：

    • 结构化方法（包括结构化分析、结构化设计和结构化编程）
    • JACKSON 方法
    • SADT／SREM

软件开发自动工具：

    • 编程工具（编辑、编译、连结程序，开发环境等）
    • 测试工具（如 Debug等）
    • 分析设计工具（PSL／PSA、AIDES、SDL／PAD）
