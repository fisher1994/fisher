第五章 编程方法
5.3 SP方法与编程
    • 结构化程序：时间与空间顺序的一致性
    • 可由机器生成
    • 可简化程序证明
    • 易于阅读和修改（可维护性）

5.4 程序内部文档（注释）

可以分为序言性注释和描述性注释
    • 序言性注释出现在程序的开头，主要描述程序的功能、界面、重要变量及其他重要信息（如开发历史、版本说明等）
    • 描述性注释出现在程序中间，说明程序语句和程序段的功能或数据的状态

5.5 编程风格

编程的一般准则：
    • 变量名：容易理解和记忆，避免混淆和歧义
    • 表达式：减少中间变量，使用括号
    • 程序逻辑简单和直接
    • 避免随意跳转（GOTO）

5.6 程序效率

提高程序效率的根本途径在于设计阶段选择良好的算法和数据结构，而不是在编程时作巧妙处理。
