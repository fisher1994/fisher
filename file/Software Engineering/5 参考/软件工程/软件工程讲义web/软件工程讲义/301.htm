第三章 需求分析和规格说明

3.1 基本概念

软件开发的两大阶段：

    • 确定问题：准确地理解问题（需求分析）和表述出来（规格说明）
    • 解决问题：系统设计和编程

需求分析阶段的基本任务：
    • 将用户对系统的要求（包括功能、性能、可靠性、安全性、费用、时间、资源等方面）用规范的方式写成一份文档DD需求说明书。

需求说明书的作用：

    • 项目开发合同
    • 设计程基础
    • 系统测试指南
    • 竣工验收依据

需求说明书的编写要求：

    • 完整性要求
    • 一致性要求
    • 精确性要求（无二义性）
    • 易于维护

系统分析员的作用：

    • 组织和协调用户与开发人员的交流
    • 不应代替用户提出要求
    • 深入了解软件开发和用户业务
