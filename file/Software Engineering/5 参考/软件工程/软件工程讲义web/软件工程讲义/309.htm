3.6 SA方法的基本步骤

本章第二节提出SA方法的基本步骤有下面四步：
  1. 获得当前系统的具体模型
  2. 抽象出当前系统的逻辑模型
  3. 建立目标系统的逻辑模型（主要工作）
  4. 作必要的补充和完善

现在我们分别讨论这几个步骤。第一步和第二步很简单，可以省略。重点要讨论的是第三和第四步。

第三步：这一步又可以分为三步进行。

1)．确定变化的范围。针对当前系统的数据流图检查每一个基本加工，如果某个基本加工在目标系统中发生了变化，则应该属于变化的范围。

2)．将属于变化范围内的基本加工合并为一个加工，确定其输入和输出数据流。

3)．对这个加工进行分解，使分解出来的各部分之间联系最少。

第四步：这一步主要从以下几方面进行考虑：

1)．人机界面

2)．系统初始化和出错处理

3)．其他方面

3.7 小结

SA方法的特点：

由往下逐层分解、图形方式描述、从现实系统模型推出目标系统模型。

SA方法的局限性：
  • 与数据库的衔接性差
  • 人机交互方式不易表达
  • 不适用于实时系统
  • 书面模型（不能上机验证）
